# Висновки

Жадібний алгоритм vs Динамічне програмування (задача про монети)

Набір монет: `[50, 25, 10, 5, 2, 1]`

> Всі результати вимірюються у **секундах** на **100 000 запусків**.

> `n` — сума,  
> `s` — кількість монет для цієї суми,  
> `m` — кількість номіналів монет.

### Результати вимірювань

| Алгоритм | Час (с) | Складність |
|---|---------|---|
| Жадібний | 0.08    | O(s·m) |
| Динамічне програмування (без кешу) | 4.1     | O(n·m + s) |
| Динамічне програмування (попередній розрахунок до 1000) | 0.05    | O(s) |

### Висновки

- **Жадібний алгоритм** (0.08 с) набагато швидший за динамічне програмування (4.1 с), якщо той постійно перераховує таблицю.
- Якщо ж **заздалегідь прорахувати** таблицю до певного ліміту (у моєму прикладі — до 1000), то динамічний алгоритм стає приблизно на **40% швидшим** за жадібний (0.05 с проти 0.08 с).
- Причина: складність динамічного алгоритму без кешу — `O(n·m + s)`, з попереднім розрахунком — `O(s)`, тоді як у жадібного — `O(s·m)` у найгіршому випадку.
